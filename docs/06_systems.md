## 入力定義（キーボードのみ・コア制約の担保）

キーは直書きせず、Godot の **InputMap のアクション名**で参照する（リマップ・制約担保のため）。
下表は既定割り当て（**要確認**: リマップ可否、対象選択の確定方式）。

### 戦闘中

右手＝移動、左手＝スキル＋パリィ、の両手レイアウト。

| アクション名 | 既定キー（候補） | 用途 |
|---|---|---|
| `move_up/down/left/right` | 方向キー or j/i/k/l or h/j/k/l（右手・要確定） | マス移動（1キー=1歩・移動歩数まで）・対象カーソル移動 |
| `face` | **Q**（左手） | 向き変更モード（→ 左右キーで左/右に向く） |
| `end_turn` | **W**（左手） | 行動終了（パス）。スキル未使用で終えると次ATBは20%から |
| `skill_a` / `skill_s` / `skill_d` / `skill_f` | A / S / D / F（左手） | 編成で割り当てた4スキルの発動（通常攻撃もスキル） |
| `parry_attack` / `parry_skill` | **R / E**（左手） | R=攻撃パリィ／E=スキルパリィ。独立リアルタイム |
| `confirm` | Enter / Space | 決定・ターゲット確定 |
| `cancel` | Esc / Backspace | キャンセル・戻る |
| `pause` | P / Esc | ポーズ |

> 左手レイアウト: 上段 `Q W E R`（向き／終了／スキルパリィ／攻撃パリィ）＋下段 `A S D F`（スキル4枠）。

**ターン入力フロー（確定方針）**: `移動 → スキル → 方向 or ターゲット`。向き変更は任意で `移動 → q → 左右`。
入力は**コマンドスタック方式**（押したキーを積む。格ゲーのプラクティス式）で受ける（詳細 [11_battle-spec.md](11_battle-spec.md) §2.5）。
1. 移動: 移動キーで位置取り（**1キー=1歩**・**移動歩数 `move_steps`** ぶんまで。`→←→`=3歩）。
2. （任意）向き変更: `q` → 左右キーで左/右へ向く。`移動 → q → 左右` で向きだけ変える、`移動 → q → 左右 → スキル` も可。
3. スキル選択: a/s/d/f でスキルを選ぶ（クールタイム中・マナ不足は不可）。**通常攻撃もこの枠**。
4. 方向 or ターゲット指定（スキル種別による）:
   - 方向系（直線・前方など）→ **左右**で方向指定（基本は自動で放つ方向に向く）。
   - ターゲット系/地点系 → 移動キー（カーソル）で敵 or マスを送り、**起動したスキルキー(a/s/d/f)を再押しで確定**（例: `a` で選択→`a` で発動）。`confirm`(Enter/Space) も併用可。
   - 全体・自己など**向き・対象指定が不要なスキルはこの手順をスキップ**。
5. `w` でいつでも**行動終了（パス）**。スキル未使用なら次ATBは20%スタート。

> 向きは**左右のみ**（上下なし、絵は左右反転）。移動とターゲットでカーソルを兼ねるため「移動フェーズ／向き／ターゲットフェーズ」を画面で明示する（要・UI 設計）。
> jikl と hjkl のどちらにするか、矢印との併用可否は要確定。

## 主要システム（公開インターフェイスの当たり・確定は実装時）

| システム | コード名（案） | 役割 / 公開する契約 |
|---|---|---|
| ATB 管理 | `ATBSystem` | 各メンバー/敵の `wait_time` を進め、満了で「ターン開始」を通知（シグナル）。 |
| パリィ判定 | `ParrySystem` | 飛来攻撃ごとに `parry_nav`（パリィナビ）を生成・縮小し、入力タイミングを判定。成功/失敗を通知。独立動作。 |
| 戦闘進行 | `BattleManager` | グリッド・メンバー・敵・勝敗を統括。各システムを仲介（疎結合のハブ）。 |
| スキル | `SkillSystem` | スキルの種別（`area_skill`/`target_skill`）・`cooltime`・効果を解決。 |
| 共有データ | `GameState`（オートロード） | 編成・進行・選択ノード・設定を保持。シーン間共有の単一窓口（→ 04_data-save）。 |

> 命名は用語集（[README.md](README.md)）に合わせる。確定した API/イベント名はここに追記して正典化する。

## 要確認

- 移動キーの確定（方向 / jikl / hjkl）と、矢印併用の可否・リマップ対応。
- 移動フェーズ／ターゲットフェーズの画面表示（カーソル兼用のため明示が要る）。
- パリィの判定幅・同時着弾時の入力解決（**多対多の難度。シビア寄りで調整**＝簡単すぎ防止）。
- ATB と敵行動の関係（敵も ATB か、別ロジックか）。
