## 入力定義（キーボードのみ・コア制約の担保）

キーは直書きせず、Godot の **InputMap のアクション名**で参照する（リマップ・制約担保のため）。
下表は既定割り当て（**要確認**: リマップ可否、対象選択の確定方式）。

### 戦闘中

右手＝移動、左手＝スキル＋パリィ、の両手レイアウト。

| アクション名 | 既定キー（候補） | 用途 |
|---|---|---|
| `move_up/down/left/right` | 方向キー or j/i/k/l or h/j/k/l（右手・要確定） | 移動範囲内でのマス移動・対象カーソル移動 |
| `skill_a` / `skill_s` / `skill_d` / `skill_f` | A / S / D / F（左手） | 編成で割り当てた4スキルの発動 |
| `parry_attack` / `parry_skill` | **R / E**（左手） | R=攻撃パリィ／E=スキルパリィ。独立リアルタイム |
| `confirm` | Enter / Space | 決定・ターゲット確定 |
| `cancel` | Esc / Backspace | キャンセル・戻る |
| `pause` | P / Esc | ポーズ |

**ターン入力フロー（確定方針）**: `移動 → スキルキー → ターゲット` の順で進む。
1. 移動: 移動キーで移動範囲内に位置取り（確定 or スキルキー押下で次へ）。
2. スキル選択: a/s/d/f でスキルを選ぶ（クールタイム中は不可）。
3. ターゲット指定: 地点攻撃ならマス、ターゲット攻撃なら敵を、移動キー（カーソル）＋`confirm` で決めて発動。

> 移動とターゲットでカーソルを兼ねるため、いま「移動フェーズか／ターゲットフェーズか」を画面で明示する（要・UI 設計）。
> jikl と hjkl のどちらにするか、矢印との併用可否は要確定。

## 主要システム（公開インターフェイスの当たり・確定は実装時）

| システム | コード名（案） | 役割 / 公開する契約 |
|---|---|---|
| ATB 管理 | `ATBSystem` | 各メンバー/敵の `wait_time` を進め、満了で「ターン開始」を通知（シグナル）。 |
| パリィ判定 | `ParrySystem` | 飛来攻撃ごとに `parry_ring` を生成・縮小し、入力タイミングを判定。成功/失敗を通知。独立動作。 |
| 戦闘進行 | `BattleManager` | グリッド・メンバー・敵・勝敗を統括。各システムを仲介（疎結合のハブ）。 |
| スキル | `SkillSystem` | スキルの種別（`area_skill`/`target_skill`）・`cooltime`・効果を解決。 |
| 共有データ | `GameState`（オートロード） | 編成・進行・選択ノード・設定を保持。シーン間共有の単一窓口（→ 04_data-save）。 |

> 命名は用語集（[README.md](README.md)）に合わせる。確定した API/イベント名はここに追記して正典化する。

## 要確認

- 移動キーの確定（方向 / jikl / hjkl）と、矢印併用の可否・リマップ対応。
- 移動フェーズ／ターゲットフェーズの画面表示（カーソル兼用のため明示が要る）。
- パリィの判定幅・同時着弾時の入力解決（**多対多の難度。シビア寄りで調整**＝簡単すぎ防止）。
- ATB と敵行動の関係（敵も ATB か、別ロジックか）。
