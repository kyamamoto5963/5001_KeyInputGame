# 06 システムと公開インターフェイス ★契約レイヤ

最終更新: 2026-06-18
状態: ドラフト（入力定義を中心に。API/イベントは実装着手時に確定）

> ここは「他シーン/他システムが知っていればよい契約」だけを小さく書く。実装詳細は各スライス側に置く。

## 入力定義（キーボードのみ・コア制約の担保）

キーは直書きせず、Godot の **InputMap のアクション名**で参照する（リマップ・制約担保のため）。
下表は既定割り当て（**要確認**: リマップ可否、対象選択の確定方式）。

### 戦闘中

| アクション名 | 既定キー | 用途 |
|---|---|---|
| `move_up/down/left/right` | 方向キー（or W/A/S/D 検討） | 移動範囲内でのマス移動・対象カーソル移動 |
| `skill_a` / `skill_s` / `skill_d` / `skill_f` | A / S / D / F | 編成で割り当てた4スキルの発動 |
| `parry_r` / `parry_e` | R / E | 攻撃パリィ／魔法パリィ（割当ては要確認）。独立リアルタイム |
| `confirm` | Enter / Space | 決定・ターゲット確定 |
| `cancel` | Esc / Backspace | キャンセル・戻る |
| `pause` | P / Esc | ポーズ |

> 移動キーと対象選択（地点/ターゲット）の操作が衝突しないよう、選択モードの入り方は実装時に確定（要確認）。
> 方向キーと W/A/S/D の併用可否も要確認（A/S/D/F をスキルに使うため、移動は方向キー主体が無難か）。

### メニュー（タイトル・編成・マップ・リザルト）

| アクション名 | 既定キー | 用途 |
|---|---|---|
| `ui_up/down/left/right` | 方向キー | カーソル移動・ノード選択・割り当て先選択 |
| `confirm` | Enter / Space | 決定 |
| `cancel` | Esc | 戻る |

## 主要システム（公開インターフェイスの当たり・確定は実装時）

| システム | コード名（案） | 役割 / 公開する契約 |
|---|---|---|
| ATB 管理 | `ATBSystem` | 各メンバー/敵の `wait_time` を進め、満了で「ターン開始」を通知（シグナル）。 |
| パリィ判定 | `ParrySystem` | 飛来攻撃ごとに `parry_ring` を生成・縮小し、入力タイミングを判定。成功/失敗を通知。独立動作。 |
| 戦闘進行 | `BattleManager` | グリッド・メンバー・敵・勝敗を統括。各システムを仲介（疎結合のハブ）。 |
| スキル | `SkillSystem` | スキルの種別（`area_skill`/`target_skill`）・`cooltime`・効果を解決。 |
| 共有データ | `GameState`（オートロード） | 編成・進行・選択ノード・設定を保持。シーン間共有の単一窓口（→ 04_data-save）。 |

> 命名は用語集（[README.md](README.md)）に合わせる。確定した API/イベント名はここに追記して正典化する。

## 要確認

- 対象選択（地点/ターゲット）の操作フロー（移動キーと兼用か、選択モードを分けるか）。
- リマップ対応の有無と範囲。
- パリィの判定幅・同時着弾時の入力解決（多対多の難度）。
- ATB と敵行動の関係（敵も ATB か、別ロジックか）。
