## このタイトル固有の適応

- **コア制約 = 入力デバイスをキーボードに限定**。これがこのタイトルの「個性」であり、
  全設計判断は「キーボードだけで完結するか」を基準に評価する。
- **狙う快感 = 先行入力の「俺できてる感」**（なぜキーボードなのか、の答え）。
  PC操作に慣れると、手が覚えたキー列を**システムの演出/処理が追いつく前に先に打ち切って確定できる**
  気持ちよさがある（例: `Win → ↓ → Enter` でシャットダウンを“目より速く”叩き込む感覚）。
  この「正確・素早い入力で先回りして処理を完了させる手応え」を戦闘のコア体験に据える。
  - 設計判断の基準に加える: **演出待ちで入力をブロックしない／手が走った分だけ先に確定できる**こと。
  - 具体機構は先行入力（予約） → [11_battle-spec.md](11_battle-spec.md) §2.5。テンポは速くしすぎない（§7・[02](02_core-loop.md)）。
- **シンプル基調＋こだわり演出を1つ**: 演出は「入力に呼応する手応え（ヒットストップ等）」を第一候補
  （→ [02_core-loop.md](02_core-loop.md)）。コアが面白くなってから足す。
- **縦スライス優先**: ステージを増やす前に、代表1ステージを一気通貫で遊べる状態にして手触り検証。
- エンジン・拠点依存の値は固定せず、選定・確認のつど docs に記録する。
