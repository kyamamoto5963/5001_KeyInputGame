# 01 要件（ヒヤリングの正本）

最終更新: 2026-06-18
状態: ドラフト（ヒヤリング2回反映。ジャンルを ATB タクティクス＋音ゲーパリィ＋分岐探索へ更新）

## 背景・課題（なぜ作るのか）

- 「**マウスもパッドも使わず、キーボードだけで操作が完結するゲーム**」を作りたい、という発想が起点。
  タイピングゲームという意味ではなく、**入力デバイスをキーボードに限定する**こと自体を面白さ・個性の核に置く。
- キーボードは「正確に・素早く押す」ことに向いた入力装置。その特性を活かし、
  **正確で素早いキー入力そのものの快感**を中心体験にしたい。

## 目的 / コアの面白さ

- グリッド上の **ATB（FF式タイムバー）**でパーティを動かしつつ、敵の攻撃には**独立リアルタイムの音ゲー式パリィ**でタイミングよく応じる。
- 快感の核は2つ: ①**パリィのタイミング成功**（リアルタイムの緊張＝正確・素早い入力）、②**ATBの好機にスキルを的確に当てる**（位置取り＋種別＋クールタイム管理）。
- 詳細は [02_core-loop.md](02_core-loop.md)。

## スコープ

### やること

- キーボードのみの操作で完結する、**3×10 共有グリッド上の戦闘**。
- **パーティ最大3人**。各メンバーは固有の**移動範囲**を持つ。
- **ATB**: ウェイトタイムが溜まったメンバーのターンに、移動（移動範囲内）＋スキル発動。
- **スキル4つ**を編成画面で **a/s/d/f** に割り当て。各スキルは**クールタイム**と**攻撃種別（地点攻撃／ターゲット攻撃）**を持つ。
- **パリィ（r/e）**: 音ゲー式（縮むわっかを円に合わせて押す）。**独立リアルタイム**で、ターン入力中でもウェイト中でも、敵の攻撃に合わせて発動可。
- **敵**: 雑魚もボスも、1体のことも複数のことも（多対多になりうる）。
- **スレスパ式の分岐探索**: 探索マップをノードで分岐しながら奥へ進む。
- **ステージ／レベル制**で拡張。後から増やす。
- PC（ダウンロード／Steam 等）向け配信を想定。

### やらないこと（現時点）

- マウス・ゲームパッド前提の操作（コア制約に反する）。※メニューのマウス補助可否は要確認。
- タイピング（単語入力）をコアにすること。
- オンライン対戦・マルチプレイ（当面）。
- 最初から大規模化（大量ノード・ストーリー作り込み）。まず縦スライス1本。
- **デッキ構築／カード収集**（スキルは固定プールを編成で割り当てる方式。デッキ構築はしない）。

## 要件（各に Done の定義）

| ID | 要件 | 受け入れ条件（Done） |
|---|---|---|
| R-01 | キーボードのみで全操作が完結 | マウス・パッドに触れず、タイトル→編成→探索→戦闘→リザルトまで遊べる |
| R-02 | ATB でターンが回る | 各メンバーのウェイトが時間で溜まり、満了で操作可能になる。視認できる |
| R-03 | ターンで移動＋スキル（両方） | 満了メンバーが「移動→スキルキー→ターゲット」の順で、移動範囲内に位置取りしスキルを発動できる |
| R-04 | スキルの種別が機能 | 地点攻撃はマスを、ターゲット攻撃は敵を対象に当たる。クールタイム制約が効く |
| R-05 | パリィが独立リアルタイムで効く | 敵攻撃に対し R=攻撃パリィ／E=スキルパリィでタイミング押下→成功でノーダメージ。ターン中/ウェイト中いつでも可。判定はシビア寄り |
| R-06 | パリィ演出が破綻しない | アニメは他アクション優先／ウェイト中はパリィモーション、という優先で見た目が破綻しない |
| R-07 | 多対多の戦闘に勝てる | 複数の敵を相手に、勝敗判定が出る |
| R-08 | 分岐探索で進める | スレスパ式マップでノードを選び、戦闘を経て奥へ進める |
| R-09 | 1スライスを遊び切れる | タイトル→編成→最小マップ→1戦闘→リザルト→再挑戦/タイトルが一気通貫 |
| R-10 | 「正確・素早い入力の快感」が出る | 触って、パリィとスキル発動が速く・気持ちよいと感じる（プレイテストで判断） |

## 想定プレイヤー・プレイの流れ

- 想定プレイヤー: キーボード操作に慣れた PC ユーザー。タクティクス＋手応えのある防御を楽しみたい層。
- プレイの流れ（縦スライスの一気通貫）:
  1. タイトル画面（キー操作で開始）
  2. 編成画面（メンバー選択＋スキルを a/s/d/f に割り当て）
  3. 探索マップ（スレスパ式に分岐、ノードを選ぶ）
  4. 戦闘（ATB で行動＋リアルタイムでパリィ。多対多をさばく）
  5. クリア／失敗 → リザルト
  6. 次ノードへ進む / 再挑戦 / タイトルへ

## 要確認リスト（未確定事項）

- グリッド寸法は 3×10 で確定か（行×列の向き、味方/敵の初期配置）。
- 移動範囲の既定値（メンバーごとの差）。※ターンは「移動＋スキル両方／移動→スキル→ターゲットの順」で確定。
- 移動キーの確定（方向 / jikl / hjkl）。
- スキルの具体セット（数値・効果・クールタイム）と、地点/ターゲットの当たり判定。
- パリィの判定幅・成功時効果（ノーダメ以外の見返りの有無）。※r=攻撃パリィ／e=スキルパリィ、シビア寄りで確定。
- 多対多でのパリィ難度調整（同時着弾の扱い、捌ききれない時の救済）。簡単すぎ防止が優先。
- 探索マップの構造（ノード種別、報酬、ローグライト性＝周回/喪失の有無）。※全体ビジョンのヒヤリング後に詰める。
- 敵の種類・出現・勝敗条件（全滅／生存／撤退）。
- UI 言語・収益方針・セーブ内容。
- タイトル名（プロジェクト名は 5001_KeyInputGame、ゲームタイトルは未定）。
