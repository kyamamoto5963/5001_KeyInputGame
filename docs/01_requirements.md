# 01 要件（ヒヤリングの正本）

最終更新: 2026-06-18
状態: ドラフト（初回ヒヤリング反映済み）

## 背景・課題（なぜ作るのか）

- 「**マウスもパッドも使わず、キーボードだけで操作が完結するゲーム**」を作りたい、という発想が起点。
  タイピングゲームという意味ではなく、**入力デバイスをキーボードに限定する**こと自体を面白さ・個性の核に置く。
- キーボードは「正確に・素早く押す」ことに向いた入力装置。その特性を活かし、
  **正確で素早いキー入力そのものの快感**を中心体験にしたい。

## 目的 / コアの面白さ

- グリッド上をキー操作でキビキビ動かし、**固定の技セットを正確・素早く出して**敵をさばく手応えを楽しむ。
- 「考える」より「**手が覚えた操作を正確に・速く出せる**」快感が主役（リズム／反射神経寄り）。
- 詳細は [02_core-loop.md](02_core-loop.md)。

## スコープ

### やること

- キーボードのみの操作で完結する**グリッド型リアルタイムアクション**。
- エグゼ型の**マス移動**（自陣エリアをマス単位で移動）。
- **固定の技セット**を各キーに割り当て、使いこなして戦う。
- **複数の雑敵**を相手にする戦闘。
- **ステージ／レベル制**で、クリアすると次へ。後から増やして拡張する。
- PC（ダウンロード／Steam 等）向けの配信を想定。

### やらないこと（現時点）

- マウス・ゲームパッド前提の操作（コア制約に反するため）。※メニューのマウス補助可否は要確認。
- タイピング（単語入力）をコアにすること。
- オンライン対戦・マルチプレイ（当面）。
- 大規模化（大量のステージ・ストーリー作り込み）を最初からやること。まず縦スライス1本。
- デッキ構築／カード収集（**当面は固定技で作る**。面白ければ後から検討 → backlog）。

## 要件（各に Done の定義）

| ID | 要件 | 受け入れ条件（Done） |
|---|---|---|
| R-01 | キーボードのみで全操作が完結する | マウス・パッドに一切触れず、タイトル→プレイ→リザルトまで遊べる |
| R-02 | グリッド上をマス移動できる | 方向キー（または相当キー）で自陣エリアをマス単位で移動。移動が「正確・キビキビ」と感じる |
| R-03 | 固定の技をキーで出せる | 割り当てたキーで技が発動。クールダウン等の制約が視認できる |
| R-04 | 複数の雑敵をさばける | 同時に複数の敵が出現し、技で倒せる。被弾でダメージを受ける |
| R-05 | 1ステージを遊び切れる | 開始→敵を全部さばく（またはウェーブ完了）→クリア/失敗の判定が出る |
| R-06 | 結果が出てやり直せる | リザルト表示後、再挑戦またはタイトルに戻れる |
| R-07 | 「正確・素早い入力の快感」が出る | 触って、入力への反応が速く・気持ちよいと感じる（プレイテストで判断） |

## 想定プレイヤー・プレイの流れ

- 想定プレイヤー: キーボード操作に慣れた PC ユーザー。短時間で手応えを得たい層。
- プレイの流れ（縦スライスの一気通貫）:
  1. タイトル画面（キー操作で開始）
  2. ステージ開始 → グリッド上で雑敵のウェーブをさばく
  3. クリア／失敗の判定 → リザルト
  4. 再挑戦 or タイトルへ
