## ADR-001: ゲームエンジンに Godot 4 を採用

- **状況**: PC（ダウンロード/Steam）向けの 2D グリッド型リアルタイムアクション。操作はキーボードのみ。
  シンプル基調・個人開発・ステージ制で拡張、という要件。
- **決定**: **Godot 4** を採用する。
- **理由**:
  - 無料・オープンソースでライセンス課金や規約変更リスクがない（個人開発・シンプル基調と整合）。
  - 2D が本職で、グリッド型2Dアクションに向く。軽量でイテレーションが速い。
  - Windows 書き出しが容易、Steam 連携（Steamworks アドオン）も実績あり。
  - 入力系（InputMap）が扱いやすく、キー割り当て・リマップが標準機能。**キーボードのみ**というコア制約と直結。
- **却下案**:
  - **Unity**: 資産・情報は豊富だが重め・C#・ライセンス考慮が必要で、今回の規模にはオーバースペック。
  - **Web系（Phaser/TS）**: キーボードと好相性だが、Steam ネイティブ配信には Electron 等のラップが必要で重くなる。
- **影響**: 以降のプロジェクト構成・入力定義（06_systems）・配信（07_roadmap）は Godot 4 前提で書く。

## プロジェクト構成（着手時に確定）

- Godot プロジェクト（`project.godot`）をリポジトリ直下 or `game/` 配下に作成（着手時に決定）。
- シーン分割・疎結合（直接参照しない）を最初から：シーン間はシグナル／オートロード（共有データ・サービス）経由。
- 入力はキー直書きせず **InputMap のアクション名**で参照（リマップ・コア制約の担保）。
