## 0. 設計原則（状態遷移を安全に保つ）★最重要

過去に最も苦労したのは**正常遷移ではなく「未定義の状態遷移」**（例: 詠唱中に吹き飛ばされた／魔法発動中に戦闘終了）。
これを構造で潰すため、以下を全機能で守る。

1. **状態を2層に分けて持つ**:
   - **ゲーム進行状態（全体）**: `進行中 / ポーズ / 終了処理中`。危険地帯は**終了**と**ポーズ**（開始は低リスク）。
   - **アクター状態（各ユニットのFSM）**: 下記「状態一覧」。
2. **状態遷移図で考える**。取りうる状態を**列挙**し、**「状態 × イベント」の全マスに挙動を明示**する。
   遷移しても**何もしない場合は「明示的に何もしない（no-op）」と書く**。**空欄＝未定義＝バグ**。
3. **行動コンテキストは1つの持ち主が必ずキャンセル・後始末できる**単位で持つ。
   戦闘終了・強制移動が来たら、その持ち主が確実に畳む。
4. **止められる時計（pausable clock）を1本通す**。ATB・クールタイム・パリィのわっか・床効果・アニメは
   **全部この時計経由**で進める（生の経過時間を直接見ない）。ポーズで全部まとまって止まる。
5. **新しい仕掛けを足すたびに問う**: 「これが**誰かの行動中／ポーズ中／戦闘終了の瞬間**に発火したらどうなる？」を実装前に決める。

## 1. 戦闘の流れ（外枠）

```
遭遇演出（「敵に遭遇した！」）→ さっと終了
  → ATB バー始動（戦闘本編。pausable clock で進行）
  → 終了判定
      ├─ 勝利／条件達成 → リザルト画面 → マップシーンへ切替
      └─ 敗北／条件失敗 → リトライ（同じ戦闘をやり直し）
```

- ポーズはいつでも可（進行中状態 → ポーズ状態）。時計が止まり全エフェクトが凍結。

### 終了条件

| 種別 | 条件 |
|---|---|
| 勝利 | 敵陣営の全滅 |
| 敗北 | 味方陣営の全滅 → **リトライ** |
| 特定条件 達成 | 例: 対象を守り切る／**N時間生存**（リアルタイム）／特定の敵だけ討伐 |
| 特定条件 失敗 | 上記の失敗（守る対象が落ちた 等）→ **リトライ** |

> 敗北・失敗は積み上げ型と整合する**リトライ**で救済（全ロストにしない）。

### 開始時のATB初期差（先手・後手）

- ATB の開始位置はユニットごとに差がある。要因: **キャラ差／スキル（不意打ち）／奇襲した・された**。

## 1.5 ATB（タイムバー）

**本質はリアルタイム。時間は止まらず流れ続ける**（止まるのはポーズのみ）。

- **敵**: ATB が満了したら**自動で行動し続ける**（AI）。プレイヤーの入力を待たない。
- **自キャラ**: ATB 満了で**アクティブ**になり、**指示があるまで待機**（行動しない）。
  - 複数が満了したら**複数がアクティブのまま待機**（例: 3人溜まれば3人待機）。
  - **早く指示しないと損**: 待機中も時間は流れ、敵は攻撃し、味方はテンポを失う。これが「素早い入力」のプレッシャー源。
- **溜まる速さ**: **キャラステータス＋スキル＋装備**に依存。
- **フォーカス（操作対象の回り方）**: アクティブになった**溜まった順**に1人ずつ自動でフォーカス。
  - フォーカス中キャラの指示（移動→スキル→ターゲット）が済むと、次に溜まっているアクティブへフォーカスが移る。
  - **同時満了の並び順は内部管理順**（決定論的）。
- 行動を終えたら**ウェイト化**して再び ATB 蓄積（ウェイト）に戻る。**ATB再開位置は終わり方で変わる**:
  - **スキルを使った**ターン → ATB **0% から**。
  - **スキル未使用で終了**（移動だけ／様子見で `w` パス）→ ATB **20% から**（早く戻れる＝無駄撃ちを強要しない）。
- 行動終了は **`w`（パス）**。アクティブ中いつでも押せる。
- **パリィは常時ライブ**（時間が止まらないため、待機中・操作中・行動中いつでも判定が走る）。

> ⚠️ 「アクティブのまま待機」「フォーカスが複数を渡り歩く」は**状態×イベント表（§5）に効く**:
> 例）フォーカス中キャラが吹き飛ばし/スタンを受けたら入力はどうなる？ 別のアクティブへフォーカスを移すか。要・表に反映。
> 可視化（個別タイムバー／円グラフ／1本のバーに敵味方アイコン）は**視認性次第で実機を見て決定**（要・実機確認）。

## 2. 盤面（フィールド）

- **横10 × 縦3** の共有グリッド（味方・敵が同じ盤を使う）。
- 移動は**盤面どこでも**可（敵陣にも踏み込める）。1ターンの移動量は**移動範囲**まで（メンバー固有）。
- **同じマスには止まれない**（押し出しは**しない**＝埋まってるマスには入れないだけ）。
- **通り抜けは常にあり**（移動の途中で他ユニットのマスを通過してよい。止まれないだけ）。
- **完全同時に同じマスを取り合った場合**の解決: **サイズが大きい方を優先 → 同サイズはキャラの内部管理順**（決定論的）。
- **キャラサイズ**: `1×1` と `2×2` が混在。**Phase 0 は 1×1 のみ**で実装し、データ構造は最初からサイズを持たせる。**2×2 は Phase 1** で差す。

### マス属性（種類は順次追加）

| 属性 | 効果（暫定） |
|---|---|
| 回復床 | 乗っている間/通過時に回復（タイミングは要決定） |
| ダメージ床 | 同上でダメージ |
| 障害物 | 移動不可（通り抜けも不可） |
| 属性強化 | そのマスのユニットの属性/威力を強化 |
| ワープ罠 | 踏むと別マスへ強制ワープ（下記強制移動ルール） |

## 3. 強制移動ルール（割り込みイベント）

強制移動は**割り込みイベント**。状態×イベント表（§5）に必ず反映する。

- **ワープ（ワープ罠など）**: 指定先へ移動。**行き先が埋まっていたら、最寄りの空きマスにランダム**で配置。
- **吹き飛ばし**: `from → to`（意図する着地）の経路を、**`to` 側から `from` に向かって逆算**し、
  **最初に乗れるマス**へ着地。乗れるマスが無ければ**動かない（その場に留まる）**。
- 2×2 ユニットの強制移動は footprint（4マス）で空き判定（Phase 1 で詳細化）。

## 4. アクター状態一覧（暫定・FSM）

> スキル/詠唱の確定に合わせて精緻化する。まずは骨組み。

| 状態 | 説明 |
|---|---|
| `ウェイト` | ATB蓄積中。ターンが回るのを待つ（時計で蓄積）。 |
| `アクティブ` | ATB満了。指示待ちで待機（まだフォーカスされていない場合も含む）。 |
| `フォーカス` | 操作中。入力を受ける（移動→スキル→ターゲットの順）。アクティブのうち1人だけ。 |
| `移動中` | 移動アニメ中。 |
| `詠唱中` | スキル発動前の溜め（割り込みに弱い区間）。 |
| `発動中` | スキルの効果適用中。 |
| `硬直` | 行動後の後隙。 |
| `被吹き飛ばし` | 強制移動アニメ中。 |
| `スタン` | 行動不能。 |
| `死亡` | 戦闘から除外。 |

> パリィは**アクター状態とは独立**に走る（リアルタイム）。アニメは「他アクション優先／ウェイト中はパリィモーション」。

## 6. スキル（データ駆動）

**スキルはコード直書きにせず、データ（パラメータの集合）として定義する**。後で**スキルエディタ**を被せられる形にする。
Godot では各スキルを**リソース（`.tres`）**として持つ想定。下の項目が「エディタで弄る項目」になる。

> **通常攻撃もスキルとして扱う**（特別扱いしない）。剣士の斬りつけ等も1つのスキルデータ。これで実装・編成・エディタが統一される。

### 盤面と向きの前提

- 盤は**横10×縦3**。狭いので**左右（長辺）が主軸**。多くのスキルは左右方向に届く。
- **向きは左右のみ**（上下なし。絵は左右反転で表現）。
- 基本、スキルは**放つ方向に自動で向く**。向き不要なスキル（全体など）は向きを参照しない。
- 手動の向き変更は **q → 左右**（入力フローは §06_systems）。

### スキルのデータ項目（エディタ対象）

| 項目 | 内容・候補値 |
|---|---|
| `id` / `name` | 識別子・表示名 |
| 対象種別 | `ターゲット攻撃`（敵を指定）／`地点攻撃`（マス指定）／`自己`／`全体`（向き・対象指定不要） |
| 射程 `range` | 使用者から届く距離（マス）。例: ファイアボール=3、兜割=1 |
| 効果範囲 `shape` | `単マス`／`直線N`／`十字`／`範囲N×N`／`列`／`行`／`全体` |
| 方向依存 | 要/不要（全体・自己は不要） |
| 詠唱 `cast_time` | 溜め時間（0＝即時）。>0 は `詠唱中` 状態を経る（割り込みに弱い） |
| コスト | **マナ消費量** ＋ **クールタイム**（両方持つ） |
| 効果リスト `effects` | 下表の効果を1つ以上。値（量・確率・持続）を持つ |
| アニメ/エフェクト | 参照（AI生成アセット）。発動時間 |

### 効果（`effects` の種類）

| 効果 | パラメータ例 |
|---|---|
| ダメージ | 量・属性（属性床と連動の余地） |
| 回復 | 量 |
| 吹き飛ばし | 距離・方向（§3 のルールで着地解決） |
| ワープ | 行き先指定（§3 のルール） |
| スタン | 時間 |
| 状態異常 | **毒**（継続ダメージ）／**混乱**（行動が乱れる）／他は順次追加。付与確率・持続 |
| バフ／デバフ | 対象ステータス・量・持続 |
| 床生成（フィールド） | 生成するマス属性（回復床/ダメージ床/ファイアフィールド等）・範囲・持続 |

### 当たり判定の方針

- **フレンドリーファイアは一旦無し**（2026-06-18）。範囲スキル・床・状態異常・吹き飛ばし等は**味方を巻き込まない**前提で進める。
  （後で「一部は巻き込む」等にする余地は残す → [09_backlog.md](09_backlog.md)。）

### 具体例（仮）

| スキル | 使用者 | 対象/射程 | 範囲 | 効果 | 詠唱 |
|---|---|---|---|---|---|
| ファイアボール | 魔法使い | ターゲット/3 | 単マス | ダメージ | 短 |
| ファイアフィールド | 魔法使い | 地点（敵足元） | 十字 | 床生成（ダメージ床）＋ダメージ | 中 |
| 兜割 | 剣士 | ターゲット/1 | 単マス | ダメージ（＋スタン?） | 即時 |

> ⚠️ 詠唱ありスキルは `詠唱中` 状態を通るため、§5 の「詠唱中 × 割り込み」を必ず埋める（過去の地雷）。

## 7. パリィ（音ゲー式・コアの快感の本体）

独立リアルタイム（§1.5）。**r=攻撃パリィ（通常攻撃用）／ e=スキルパリィ（スキル攻撃用）**。

- **パリィリング（わっか）**: 敵攻撃の**着弾の約1秒前**に出現し、円へ収束。合う瞬間に対応キー押下で**ノーダメージ**。
  - ⇒ 攻撃は最低でも**約1秒の予告時間**を持つ。これがテンポの自然なブレーキになる。**要調整**（1秒が良いかは触って判断）。
- **複数ヒット**: 1回の攻撃が複数ヒットすることあり（各ヒットにわっか／まとめるかは要確認、基本は各ヒット）。
- **判定**: **1段階のみ**（Good/Perfect 等の段階なし）。
  - **失敗（タイミング外し・誤キー）したら、最寄りのパリィチャンスが消える** → 連打・先押しを防ぐ。
- **同時着弾**: ある程度の**時間幅に収まる攻撃は1押しで同時パリィ成功**でよい。幅は**キーボードの入力受付時間**に依存（要調整）。
- **成功効果**: ノーダメージ＋追加見返り（**反撃／マナ回復／ATB加速**など）。**キャラごとにパリィ効果が異なる**。
- **連続パリィ（コンボ）**: 連続成功で**ドロップ率が上がる**（収集と接続 → 08）。
- **撃ち分け**: 種別を**間違えて押す＝失敗扱い**（最寄りチャンス消費）。※最終確認。
- **多対多の難度**: **シビア寄り**で調整（簡単すぎ防止）。ただしテンポは速すぎない（下記）。

### テンポ方針（コアの手触り）

- 目標は **LoL くらいの中速・タクティカル**。**One Step from Eden は速すぎ**と感じる＝もっと落ち着いた速度にする。
- 速すぎる手触りは好みでない。**1秒の予告わっか**がテンポを規定する装置になる。盛って速くしすぎない。

## 8. 敵AI（軽量・データ駆動）

**ビヘイビアツリーは使わない**（過去に初期化が重く難儀）。代わりに**フェーズ＋ルールリスト（decision list）を
データで持つ**軽量方式。判断は**ATB満了の瞬間だけ**走る（毎フレームでない）＝ほぼタダ、重い初期化なし。
各敵のAIは**リソース(.tres)**で定義（スキルと同じデータ駆動。後でエディタ化も可）。

### 構造

- **フェーズ**（HP閾値などで切替）。各フェーズが「抽選プール（スキルID＋重み）＋ワンショットルール」を持つ。
- 判断手順: ルールを**上から評価し、最初に当てはまるものを実行** → 無ければフェーズの抽選プールからランダム。
- 例（雛形）:
  ```
  Phase A: HP ≥ 30%  → 使えるスキルから重み付きランダム
  Phase B: HP < 30%  → 進入時に1度だけ回復(ワンショット)、以降は回復込みでランダム
  ```

### 敵の「性格」（データで振る舞いを切替）

敵の**種類・性格**ごとに以下を束ねて持つ（＝同じ仕組みで多彩に出せる）:

- **ターゲット選択方針**: 最寄り／最弱(HP低)／ランダム／**フォーカス中の味方を狙う**／など（どれもあり、敵で選ぶ）。
- **移動／フォールバック方針**: 使えるスキルが無い・射程外のとき → **接近する／距離を取る／パスする**（どれもあり、敵で選ぶ）。
- **フェーズ別の抽選プールと重み**、ワンショットルール。

### 「使えるスキル」とエッジケース（§5 の精神で必ず決める）

- 候補は**クールタイム・マナ・射程/対象が成立する**スキルのみ。
- **候補ゼロ**（全部クールタイム/マナ不足/射程外）→ 性格の**フォールバック方針**（接近/距離取り/パス）に従う。
- 対象消失・フェーズ境界・割り込み（吹き飛ばし等）は §5 状態×イベント表に準拠。

### データ項目（エディタ対象・案）

| 項目 | 内容 |
|---|---|
| `phases` | しきい値・抽選プール（スキルID＋重み）・ワンショットルール |
| `target_policy` | 最寄り/最弱/ランダム/フォーカス狙い 等 |
| `fallback_policy` | 接近/距離取り/パス |
| 攻撃種別（各スキル側） | 近接/遠隔/範囲（スキルデータに内包、§6） |
