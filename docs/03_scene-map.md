# 03 シーンマップ・遷移図 ★契約レイヤ

最終更新: 2026-06-18
状態: ドラフト

> シーン同士は直接参照しない。遷移・共有状態はシグナル／オートロード（共有データ・サービス）経由。
> 各シーンの作業では「このシーン＋契約レイヤ」だけ載せれば済むようにする。

## シーン一覧

| シーン | コード名 | 役割 |
|---|---|---|
| タイトル | `Title` | 起動直後。キー操作でゲーム開始。設定への入口。 |
| 編成画面 | `Loadout` | パーティ（最大3人）選択＋スキルを a/s/d/f に割り当て。出撃する。 |
| 探索マップ | `Map` | スレスパ式の分岐マップ。ノードを選んで進む。 |
| 戦闘 | `Battle` | グリッド戦闘。ATB＋リアルタイムパリィ。多対多を捌く。 |
| リザルト | `Result` | 勝敗・報酬表示。次ノード／再挑戦／タイトルへ。 |
| 設定 | `Settings` | 音量・キーリマップ（可否は要確認）など。 |

## 遷移図

```
Title ──開始──> Loadout ──出撃──> Map ──ノード選択(戦闘)──> Battle
  ^                                  ^                        │
  │                                  │                     勝敗判定
  │                              次ノードへ                   │
  │                                  └──────── Result <───────┘
  └────────────────── タイトルへ ─────────┘
                  （Result から 再挑戦 / 次へ / タイトル）

Title ──設定──> Settings ──戻る──> Title
```

- **Map → Battle → Result → Map** が探索のループ。マップを進み切る（ボス撃破等）と Stage クリア。
- 縦スライスでは Map を最小（数ノード）にして Battle 1回を通す。

## 各シーンが他に出す主な契約（詳細は 06_systems.md）

- `Loadout` → 共有データに「パーティ編成＋スキル割り当て」を書き、`Map`/`Battle` が読む。
- `Map` → 「選んだノード（戦闘内容）」を共有データに渡し、`Battle` が読む。
- `Battle` → 戦闘結果（勝敗・報酬）を `Result` に渡す。
- 共有データ・セーブのスキーマは [04_data-save.md](04_data-save.md)（未着手）で確定。

## 要確認

- 設定でキーリマップを許すか（キーボードのみ制約との兼ね合い）。
- 探索の途中でセーブ/中断できるか（→ 04_data-save）。
- Map のノード種別（戦闘以外に休憩・報酬・分岐イベント等を置くか）。
