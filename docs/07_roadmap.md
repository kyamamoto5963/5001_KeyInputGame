# 07 ロードマップ（開発フェーズ・マイルストーン）

最終更新: 2026-06-18
状態: ドラフト

> 原則: **縦スライス → 横展開**。盛る前に核を遊べる状態にして検証する（→ [02_core-loop.md](02_core-loop.md)）。
> 1作目の到達目標は **ワールドマップ開放まで**（→ [08_world-vision.md](08_world-vision.md)）。大規模化はしない。

## フェーズ

### Phase 0 — 縦スライス：冒頭戦闘（つかみ＝戦闘ショーケース）★最優先

記憶喪失の冒険者が**開始直後にいきなり戦闘**に入る、その1戦を端から端まで遊べるようにする。
これがゲームの核を見せる場であり、面白さ検証の本丸。

- 3×10 共有グリッド、味方1〜2人＋雑魚数体。
- ATB でターン（移動＋スキル、入力順 移動→スキル→ターゲット）。
- スキルを a/s/d/f に仮割り当て（地点/ターゲット＋クールタイム）。
- パリィ（r=攻撃 / e=スキル）を独立リアルタイムで実装、判定はシビア寄りで調整。
- 勝敗判定 → 簡易リザルト → やり直し。
- **Done**: 触って「パリィとスキルが気持ちいい」と言える。ダメなら方向転換。

### Phase 1 — 戦闘の幅と手触り

- スキルの種類・敵パターンを増やし、パーティ3人・多対多を成立させる。
- パリィの同時着弾処理・難度調整（簡単すぎ防止）。こだわり演出（パリィ成功）を1つ入れる。

### Phase 2 — マップと拠点（横展開の入口）

- エリアマップ（ポイント＋ルート）を最小構成で。中間ポイントの戦闘/イベント。
- 街ポイント（メニュー式：話を聞く/買い物/ギルド）を1つ。
- 探索ポイント1種（基本は全探索、特定条件で後戻り不可の分岐も）。

### Phase 3 — 育成・収集

- 主人公のスキル/パラメータ振り分け、教会での振り直し＋ユニークスキルのガチャ割り当て（入手済みはプールから選択可）。
- 収集（ドロップ・調合・掘り出し物）とレア武器→ビルド組み直しの遊び。
- 積み上げ型の進行・セーブ（→ 04_data-save）。

### Phase 4 — 1作目の通し（ワールドマップ開放まで）

- エリアを通しでつなぎ、エリア端→別エリア、ワールドマップ開放までを成立させる。
- ストーリーの芯を入れる（記憶を取り戻す／成り上がり）。
- リリース準備（→ [conventions.md](../../5000_GamePhilosohy/conventions.md) のリリース前チェック）。
