# 12_character-rig.md — キャラ・リグ／パーツ合成 仕様

状態: ドラフト（2026-06-27 起票）

カットアウト（操り人形）方式のキャラを、**1枚絵の切り出しではなく「共有スケルトンに乗るパーツの組み合わせ」**として
データ駆動で組み立てる仕様。キャラクリエイト（着せ替え）にそのまま伸ばせる土台を置く。

> 動機: 全身1枚絵を都度切り出すと (a) キャラごとに関節位置がブレる (b) 継ぎ目にムラが出る (c) パーツ流用できない。
> 全パーツを**同一の基準（キャンバス＋関節座標）**に従わせれば、これらが消え、量産・差し替え・色替えが効く。
> 既存の試作: [`res://scripts/cutout_character.gd`](../dev/scripts/cutout_character.gd)（`parts.json` 駆動の3パーツ版）／
> プレビュー: [`res://scenes/char_setup/`](../dev/scenes/char_setup/)。本仕様はこれを**フルボーン＋スロット**へ拡張する。

決定: 粒度は **フルボーン（腕脚分離）**。当面は**仕様の確定が先**（実装はこの文書に合意後）。

## 用語（→ README 用語集に追記）

| 日本語（正） | コード識別子（英） | 意味 |
|---|---|---|
| スケルトン | `skeleton` | 全パーツが共有する関節定義（座標＋ボーン階層＋レイヤ順）。例 `humanoid_v1`。**唯一の基準**。 |
| ボーン / 関節 | `bone` / `joint` | スケルトン上の可動点。親子で繋がりパーツがぶら下がる。 |
| バインドポーズ | `bind_pose` | パーツを描く基準姿勢（直立・腕を体から少し離した安静）。関節座標はこの姿勢で定義。 |
| スロット | `slot` | 着せ替えの差し替え枠（head/upper/lower/arm_upper_r …）。1スロット=1パーツ。 |
| パーツ | `part` | スロットに入る1枚の絵＋メタ（所属ボーン・レイヤ）。スケルトン基準で描かれる。 |
| アタッチ | `attach` | パーツがどのボーンの子になるか（ぶら下がり先）。 |
| レイヤ順 | `z_order` | パーツの前後描画順。背面パーツ→前面パーツ。 |
| ティント | `tint` | パーツの色替え（生成し直さず量産するための乗算/置換カラー）。 |
| キャラデータ | `character_data` | 「どのスロットにどのパーツ＋tint」を束ねた1キャラの定義（差し替えの実体）。 |

## 1. 正準スケルトン `humanoid_v1`

全パーツ・全キャラが従う固定基準。これを変えると全パーツの作り直しになるため、**バージョン名で固定**（破壊的変更は `humanoid_v2` を切る）。

- **共通キャンバス**: `256 x 384`（パーツ原画はこの座標系で描く。原点=左上、x中心=128）
- **接地**: `feet` を基準にowner側が配置（足元=グリッドのマス）
- **向き**: 左右のみ。`facing` 左向きは `_rig` のミラーで表現（→ 11_battle-spec §6, `art_facing` 同様の扱い）

### 1.2 ボーン階層
```
hip (root)
├─ chest ─ neck ─ head
│   ├─ bust*            （chest子・揺れボーン／任意）
│   ├─ shoulder_r ─ elbow_r ─ wrist_r
│   └─ shoulder_l ─ elbow_l ─ wrist_l
├─ tail*               （hip子・揺れボーン連鎖／任意）
├─ thigh_r ─ knee_r ─ ankle_r
└─ thigh_l ─ knee_l ─ ankle_l
```
`*` ＝二次モーション用の揺れボーン（任意。→ §5.3-jiggle）。髪・スカート・マントは専用ボーンを足さず、所属ボーン（head/hip）配下のパーツにスプリングを掛ける。

## 2. スロット＆レイヤ

キャラ＝各スロットに1パーツ。各スロットは **attach（所属ボーン）** と **z_order（描画順）** を持つ。
左右の腕脚は「奥(back)／手前(front)」で前後し、`facing` 反転時に z を入れ替える（§5.3）。

| スロット | attach | z_order | 必須 | 備考 |
|---|---|---|---|---|
| `back` | hip | 0 | 任意 | マント・背負い物（最背面）。**スイング**（→ §5.3-jiggle） |
| `tail` | hip（揺れボーン連鎖） | 2 | 任意 | しっぽ。**スイング**（→ §5.3-jiggle） |
| `hair_back` | head | 5 | ○ | 後ろ髪（体・顔より背面）。**スイング** |
| `leg_thigh_b` / `leg_shin_b` / `foot_b` | thigh/knee/ankle（奥脚） | 10 | ○ | 奥側の脚 |
| `lower` | hip | 20 | ○ | 下衣（スカート/ズボン）。腰でスイング |
| `arm_upper_b` / `arm_fore_b` / `hand_b` | shoulder/elbow/wrist（奥腕） | 25 | ○ | 奥側の腕 |
| `body` | hip | 30 | ○ | 素体（肌・胴の下地） |
| `upper` | chest | 40 | ○ | 上衣。胸でスイング |
| `bust` | bust（chest子の揺れボーン） | 42 | 任意 | 胸の独立揺れ（パーツ方式時。→ §5.3-jiggle） |
| `leg_thigh_f` / `leg_shin_f` / `foot_f` | thigh/knee/ankle（手前脚） | 45 | ○ | 手前側の脚 |
| `head` | head | 50 | ○ | 顔 |
| `hair_front` | head | 55 | ○ | 前髪（顔より前面）。**スイング** |
| `hat` | head | 60 | 任意 | 被り物 |
| `arm_upper_f` / `arm_fore_f` / `hand_f` | shoulder/elbow/wrist（手前腕） | 70 | ○ | 手前側の腕 |
| `weapon` | wrist_f（手）/ 背中 | 80 | 任意 | 武器・道具（最前面寄り）。**ミラーで両利き共用**（利き手は意識しない） |

> 注: 「奥/手前(b/f)」は左右(r/l)に**facing依存でマップ**する（右向き時 r=奥、左向き時 l=奥 など）。スロットIDは奥/手前で持ち、ボーンは左右に解決する。詳細は§5.3で確定。

## 3. パーツ仕様（ムラ対策の本体）

**保存形式は「bbox密着クロップ＋パーツごとのアンカー(anchor)」**を採用（＝最小サイズA方式）。
固定キャンバスは**生成・オーサリングの土俵**としてのみ使い、保存形式には持ち込まない。
理由: 固定キャンバスは (a) フレームをはみ出す物（長武器/大帽子/マント/しっぽ） (b) スケルトン種別追加（子供/大型/四足） (c) メモリ無駄 で**いずれ破綻する**ため。

- **オーサリング**: 共有テンプレ（256x384・関節マーク入り）上で、バインドポーズの定位置に描く/生成する（位置合わせ・AI一貫性のため）
- **bake**: そのPNGを **bbox密着クロップし、`anchor = 関節座標 − クロップ左上` を自動算出**して保存（手でアンカーを置かない）
- **anchor の意味**: パーツが回転する基準点（＝アタッチ先ボーンの関節）。組み立て時に anchor をボーン原点へ合わせる（§5.1）
- **背景透過**（PNG/RGBA）。縁の白ハロー禁止
- **均一ライティング**：強い影・リムライト・接地影は焼き込まない（動くと破綻するため）
- **隣接パーツとの重なりは最小限**：関節をまたぐ部分はわずかに重ねる（隙間防止）。胴を腕が横切らない
- **画風ロック**：全パーツ同一スタイル（同一LoRA / 固定プロンプト・seed）

### 3.1 ファイル配置・命名
```
dev/assets/char/parts/<slot>/<id>.png        # bbox密着クロップ済みPNG。例 upper/coat_witch.png
dev/assets/char/parts/<slot>/<id>.json       # { slot, attach, anchor:[x,y], z, (size) }
dev/assets/char/skeletons/humanoid_v1.json   # スケルトン定義（§1）
dev/data/characters/<name>.tres|json         # キャラデータ（§4）
```
- `id` は slot を含めず短く（slot はフォルダで分かる）。例 `coat_witch`, `skirt_witch`, `brown_bangs`, `hat_navy`。
- パーツ json の `anchor` は bake が自動で入れる（テンプレ由来）。`attach`/`z` は slot 既定から継承可（差分だけ書く）。
- **現 `parts.json` との関係**: 既存は `offset`（＝クロップ左上）と `joints` を持つので `anchor = joint − offset` で**ほぼA形式に変換可**。

## 4. キャラデータ（差し替えの実体）

```jsonc
{
  "id": "witch_lyra",
  "skeleton": "humanoid_v1",
  "parts": {
    "body": "std",
    "lower": "skirt_witch",
    "upper": "coat_witch",
    "head": "face_a",
    "hair_back": "brown_long_back",
    "hair_front": "brown_bangs",
    "hat":  "hat_navy",
    "tail": null,
    "arm_upper_f": "coat_sleeve", "arm_fore_f": "coat_cuff", "hand_f": "bare",
    "arm_upper_b": "coat_sleeve", "arm_fore_b": "coat_cuff", "hand_b": "bare",
    "leg_thigh_f": "skin", "leg_shin_f": "boot_high", "foot_f": "boot_heel",
    "leg_thigh_b": "skin", "leg_shin_b": "boot_high", "foot_b": "boot_heel"
  },
  "tint": { "upper": "#3a2f5a", "lower": "#3a2f5a" }   // 任意
}
```
- 省略スロットは「無し」（hat等）または body 既定で補完。
- `tint` は乗算/置換（実装で確定）。**色替えだけのバリエーションは生成し直さずデータで量産**。

## 5. 組み立て（CutoutCharacter の拡張）

現 `CutoutCharacter` は「parts.json の joints からボーン3つ＋スプライト3枚」を作る。これを次に拡張する。

### 5.1 ビルド手順
1. `skeleton`（humanoid_v1）を読み、**全ボーンを階層生成**（§1.2）
2. `character_data.parts` を走査し、各スロットの `id` から PNG＋パーツjson(`anchor`等)を読む
3. パーツを `attach` ボーンの子に追加。`centered=false`、**`sprite.position = -anchor`**（＝anchor画素がボーン原点に乗る）
4. `z_order` で各スプライトの `z_index` を設定
5. `tint` があれば `modulate` 等で適用

### 5.2 アニメ（手続き＋将来AnimationPlayer）
- アイドル/攻撃は**ボーン回転の手続き**（現行を踏襲）。フルボーン化で「腕振り」「歩行の脚運び」が可能に。
- 本実装では `AnimationPlayer` でボーン姿勢をキーフレーム化（idle/walk/attack/hit）する案を併走（§9）。

### 5.3-jiggle 二次モーション（揺れ・スプリングボーン）

胸・髪・スカート等の「遅れて揺れる」表現は **スプリングボーン（バネ-ダンパ）** で出す。
本体ボーンの加速度を入力に、子の揺れボーンが遅れて追従・オーバーシュートする（慣性）。

- **対象（揺れスロット）**: `bust`（chest子）／`hair_front`・`hair_back`（head子）／`lower`(スカート・hip子)／`back`(マント)／`tail`(しっぽ・揺れボーン連鎖)。
- **入力**: 親ボーンのワールド加速度（歩行の上下・着地・方向反転・攻撃リーン）。
- **パラメータ（パーツ/スケルトンに持たせる）**:
  - `stiffness`（戻る強さ）／`damping`（収束の速さ）／`max_offset`（暴れ防止のクランプ）／`squash`（上下の伸縮量）。
  - 既定は**控えめ**。やり過ぎ防止に `max_offset` でクランプ必須。
- **実装段階**:
  1. **現フェーズ（全身1スプライト）**: `chest` ボーンにバネを入れ、上下バウンス＋スカッシュで近似（専用パーツ不要）。
  2. **パーツ方式（将来）**: `bust` を chest 配下の独立揺れボーン＋専用パーツにして、胸だけを揺らす（高忠実）。髪・スカートも同様に揺れボーン化。
- **数値の安定**: 半暗黙オイラー等でフレームレート非依存に。ポーズ中（ATB停止）は揺れも止める。

> 揺れは**演出**であってゲーム判定には関与しない（当たり/向き/座標は本体ボーンが正）。

### 5.3 facing と奥/手前
- スロットは奥/手前(b/f)で持つ。`facing` により b/f → 左右(r/l)ボーン＋z入れ替えを解決。
- 全体ミラーは `_rig.scale.x` 反転（現行）。**武器はミラーで両向き共用**（利き手＝右/左は意識しない）。片手武器は「手前手に持つ」固定で良い。

## 6. AI生成パイプライン（パーツを揃える）

素の txt2img 単発はNG（位置・画風がばらける）。揃える定石：
1. **定位置テンプレ**: バインドポーズのガイド画像（薄いスケルトン or 素体シルエット）を用意
2. **領域生成**: そのスロット領域だけ **img2img / インペイント** で描かせる → 位置が常に同じ
3. **ControlNet**（pose/lineart/depth）でシルエット拘束
4. **画風ロック**: 同一LoRA or 固定スタイルプロンプト＋固定seed
5. **色替えは tint** で量産（再生成しない）

> 当面は既存 char_001/002 から切ったパーツを**仮素材**に使い、仕組み（§1〜§5）を先に固める。生成テンプレ整備はその後。

## 7. 既存からの移行

- 現 `parts.json`（1キャラ=頭/胴/スカート、joints同梱）は **char_001/002 専用の暫定**として残置可。
- 新方式は `skeletons/humanoid_v1.json` ＋ `parts/<slot>/` ＋ `characters/<name>` に分離。
- `CutoutCharacter.build()` を「skeleton + character_data」を取る形にオーバーロード（旧 parts.json 経路は当面共存）。
- `char_setup` は新方式のプレビュー（スロット差し替え・tint確認）へ拡張。

## 8. 確定事項

- 粒度: **フルボーン（腕脚分離）**。
- パーツ保存形式は **bbox密着クロップ＋アンカー（A方式）**。固定キャンバスは**生成テンプレ専用**（保存に持ち込まない）。anchor は bake が自動算出。
- キャラは**スケルトン＋スロット別パーツ＋tint**のデータで組む。
- 色バリエーションは**生成せず tint で量産**（採用確定。ブレンド方式は実装しながら決める）。
- **実装順**: まず**パーツ合成（組み立て）まで**。`AnimationPlayer` 化はその後（いずれ必要）。
- 進め方: **仕様先行**（本書合意 → 仮パーツで仕組み実装 → 生成テンプレ整備）。
- **関節座標(§1.1)は実装しながら詰める**（最初から確定させない・暫定値スタート）。
- **髪は前髪/後ろ髪を分離**（`hair_front`=顔の前 / `hair_back`=背面）。両方スイング対象。
- **weapon はミラーで両向き共用**（右利き/左利きは意識しない。片手武器は手前手固定）。
- **二次モーション（揺れ）は機能として入れる**（§5.3-jiggle）。対象に **スカート/マント/しっぽ** も含める。数値は後で調整。
- **しっぽ(`tail`)スロットを用意**（揺れボーン連鎖）。
- **スケルトンはいずれ複数種に増やす**（`humanoid_v1` からバージョン/種別で追加。子供/大型/四足等）。

## 9. 未確定・宿題

- 関節座標(§1.1)の具体値（実パーツに合わせて随時更新）。
- `tint` のブレンド方式（乗算 / パレット置換 / マスク別塗り）— 採用確定、方式は実装しながら。
- スカート/マント/しっぽ/髪の揺れの実装手段（スプリングボーン連鎖 or Bone2D変形 or 物理）。
- `AnimationPlayer` 化（idle/walk/attack/hit のクリップ化）— パーツ合成の後に着手。手続きとの併用方針。
- 生成テンプレ→bake（クロップ＋anchor算出）のスクリプト整備。
- 追加スケルトン種別の具体（どの体型から増やすか）。
```
